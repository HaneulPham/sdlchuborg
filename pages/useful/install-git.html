Git Setup and Basic Commands Guide for Mac

Step 1: Install Git on Mac

  1. Open Terminal:

    Press Command + Space, type "Terminal," and press Enter.

  2. Install Git via Homebrew (Recommended):
    1. First, install Homebrew if you haven't already. 
      /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
    2. Then, install Git using Homebrew: 
      brew install git
  3. Check if Git is Installed: 
    git --version

    You should see the installed version number, which means Git is successfully installed.

Step 2: Configure Git

  1. Set Your Username: 
    git config --global user.name "Your Name"
  2. Set Your Email: 
    git config --global user.email "your.email@example.com"

Step 3: Create a Repository

  1. Initialize a New Git Repository: 
    mkdir my-git-project
    cd my-git-project
    git init

    This command creates a new Git repository.

  2. Clone an Existing Repository (if you have one to work on): 
    git clone [repository URL]

    Replace [repository URL] with the actual URL of the repository you want to work on.

Step 4: Basic Git Commands

1. Commit Changes

Create or modify a file:

echo "Hello, Git!" > hello.txt

Add the file to staging:

git add hello.txt

Commit the changes:

git commit -m "Add hello.txt file"

2. Push Changes

If you have a remote repository (e.g., on GitHub), you can push your changes.

First, add the remote URL:

git remote add origin [repository URL]

Push your changes to the remote repository:

git push origin main

Replace main with your branch name if it’s different.

3. Pull Changes

To pull the latest changes from the remote repository:

git pull origin main

4. Merge Changes

Switch to your main branch (or another branch):

git checkout main

Merge another branch into your main branch:

git merge [branch-name]

Replace [branch-name] with the name of the branch you want to merge.

5. Rebase

Switch to the branch you want to rebase:

git checkout [branch-name]

Rebase onto another branch:

git rebase main

This moves your branch on top of the latest changes from main.

6. Push After Rebase

After rebasing, you need to force push to the remote branch:

git push -f origin [branch-name]

Step 5: Practice Scenario

To practice, you could follow this simple scenario:

  1. Create a new file, add content, and commit it.
  2. Make a change, commit again.
  3. Create a new branch (git checkout -b new-feature) and make changes there.
  4. Merge the new-feature branch back into main (git checkout main then git merge new-feature).
  5. Try rebasing (git rebase main) with another branch.
